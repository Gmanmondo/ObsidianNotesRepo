## Creating a Gant Chart
Gantt chart is a horizontal bar chart with a list of activities on the left and a timeline on the horizontal axis

Traditional Gantt chart does not show dependencies whereas modern charts add critical path and arrows to show predecessors
![[Pasted image 20260223201657.png]]

## Network Diagram
- Known as PERT or PERT/CPM chart
- Task relationships
- Used to determine critical path
- Software like Microsoft Project can be used for this
![[Pasted image 20260223201829.png]]

## Critical Path
- Path with the longest duration is called the critical path
- if any task on the critical path is delayed, the project is delayed
- Tasks not on the path have slack
![[Pasted image 20260223202012.png]]

## Slack
Amount of time a task could be delayed without delaying the project
### Total Slack
Allowable delay
### Free Slack
Allowable delay without affecting the start of any dependent
### Safety Slack
Remaining allowable delay of an activity if all predecessors finish as late as possible

## Network Diagram compared to Gantt
- ND is more technical
- ND is not to a scaled timeline
- Gantt is easy to read and explain
- Gantt is good for illustrating schedule
- Project managers depend on ND to plan and adjust
- Gantt is developed from ND for purposes of communicating the plan
- Both are easily produced from PM software

## Finalizing Project Schedule
- May have to negotiate with sponsor or adjust expectations
- Schedule will be impacted by project resource plans
